# 一、完善后完整产品需求文档（Chrome 扩展 Manifest V3
> 目标：英文划词翻译扩展，英译中，免费翻译 API、生词高亮、站点黑名单，兼容最新谷歌浏览器 Manifest V3 规范
## 核心功能清单（完善细化）
1. 划词翻译（基础能力）
鼠标选中网页英文单词 / 短语，松开鼠标短暂延迟后弹出翻译悬浮窗（可自定义延迟：0~1000ms）
触发规则：
仅匹配英文文本（过滤纯数字、符号、中文）
支持单词、短语、短句翻译
按住Ctrl强制开启划词翻译（可选快捷键触发）
悬浮窗口内容：
原文、中文释义、音标（如有 API 支持）、简单词性
【加入生词本】按钮；已收录生词显示【移出生词本】
样式：悬浮窗不遮挡文字，可跟随鼠标，超出页面边界自动避让
2. 生词本系统（核心特色）
生词存储：使用 chrome.storage.local 本地持久化（无需登录，数据存在浏览器本地）
字段设计：
json
{
  "word": "单词原文",
  "translation": "中文释义",
  "addTime": "时间戳",
  "count": "页面出现次数",
  "reviewed": "是否标记复习"
}
生词自动高亮
当前网页加载完成后，自动扫描页面文本
匹配生词本内单词，添加下划线 / 自定义高亮色（颜色可配置）
鼠标悬浮高亮生词时，弹出迷你翻译提示框（无需再次划选）
限制：不修改输入框、富文本编辑器、代码块内文本，避免冲突
生词本管理面板（扩展弹窗 / 独立页面）
列表展示所有生词：单词、释义、添加时间、出现频次
支持：单个删除、批量删除、搜索生词
导出生词（JSON / 纯文本）、导入生词
简易复习模式（可选）
3. 网站黑名单（禁用划词功能）
黑名单列表：域名匹配（如 *.youtube.com、github.com）
匹配规则：精确域名 / 通配符域名
功能逻辑：
当前页面域名在黑名单 → 关闭划词翻译、关闭生词自动高亮
扩展设置页手动添加 / 删除黑名单网址
快捷入口：悬浮窗 / 扩展图标菜单【将当前网站加入黑名单】
支持临时开关：全局一键暂停所有划词翻译
4. 翻译 API 选型（免费优先，英译中）
优先选用无 Key、免费公共接口（规避付费、配额问题）
备选清单（开发时优先测试可用性）
有道免费 API（简易接口）
DeepL 免费公开接口（限流注意）
LibreTranslate 【首选推荐】开源免费、自托管 / 公共实例，无密钥
Google Translate 简易非官方接口（注意访问稳定性）
需求约束：
请求失败友好提示（网络异常、接口限流）
增加防抖：短时间重复查询同一单词不重复请求 API
单词翻译结果本地缓存，减少 API 请求
5. 扩展设置面板
可配置项：
划词翻译：开启 / 关闭、选中延迟时长
悬浮窗口样式：字体大小
生词高亮：开启 / 关闭、高亮颜色、下划线开关
翻译 API 切换（多接口备选）
网站黑名单管理
清除本地缓存、清空生词本
6. 交互约束 & 兼容性
采用 Manifest V3（Manifest V2 谷歌已淘汰）
区分三类脚本：
content_script：注入网页，实现划词监听、文本扫描、生词高亮
service_worker（后台脚本）：统一转发翻译 API 请求，解决跨域 CORS
popup.html：点击扩展图标弹出简易面板（快速打开生词本、临时开关）
options.html：完整设置页面
安全限制：
不收集用户网页内容，所有生词本地存储，不上传第三方服务器
避免频繁 DOM 扫描造成页面卡顿，增加节流防抖
兼容：普通网页；避开 PDF 查看页、chrome 内置页面（chrome:// 无法注入脚本）

# 技术栈
HTML + CSS + Vanilla JavaScript（不引入大型框架，减小体积）
存储：chrome.storage.local
跨域请求：Service Worker代理翻译API请求，解决CORS

# 强制开发需求细则
1. 划词翻译逻辑
   1.1 content.js监听selectionchange文本选中事件，增加防抖延迟
   1.2 正则筛选英文内容，过滤中文、纯数字、符号
   1.3 选中文本去除首尾空格，长度限制1~200字符
   1.4 调用service-worker发起翻译请求，优先读取本地翻译缓存
   1.5 弹出悬浮DOM窗口，提供【加入生词本】按钮
   1.6 悬浮窗点击外部区域自动关闭；鼠标移入窗口不消失

2. 生词本与自动高亮
   2.1 页面初始化完成后，节流方式遍历页面文本节点
   2.2 匹配storage内生词列表，给文本包裹span标签添加高亮样式
   2.3 排除 <input> <textarea> <code> <pre> 标签内文本，禁止修改
   2.4 鼠标hover高亮生词，弹出小型翻译tooltip
   2.5 新增生词/删除生词后，实时刷新当前页面高亮

3. 网站黑名单机制
   3.1 content脚本启动时先获取当前域名，和黑名单比对
   3.2 黑名单命中：禁用划词监听、停止生词扫描高亮
   3.3 设置页面支持添加域名，支持【添加当前站点】快捷按钮
   3.4 域名匹配规则：包含匹配（如域名为a.com，则a.com所有子页面生效）

4. 翻译API层
   4.1 默认优先使用LibreTranslate公共实例，英译中
   4.2 实现多接口故障转移：一个API报错自动切换备选接口
   4.3 建立翻译缓存对象，key=原文，value=翻译结果，减少网络请求
   4.4 接口报错时在弹窗提示：翻译服务暂时不可用

5. 生词本页面功能
   5.1 分页展示所有生词，支持关键词搜索
   5.2 单条删除、批量清空生词
   5.3 导出生词列表为文本/JSON文件
   5.4 点击单词可快速重新查询释义

6. 设置页面功能
   6.5 划词开关、触发延迟调节
   6.6 生词高亮开关、自定义高亮颜色
   6.7 API接口切换选项
   6.8 黑名单管理列表
   6.9 一键清空缓存、清空生词本

# 性能要求
1. 页面生词扫描增加节流，滚动时不重复扫描，避免页面卡顿
2. DOM新增元素使用文档碎片优化，减少页面重绘
3. 悬浮窗DOM只创建一次，重复复用，不频繁新建销毁

# 边界异常处理
1. chrome://、edge:// 等浏览器内置页面，脚本自动不运行
2. SPA单页应用路由切换后，重新触发生词扫描
3. 网络断开时翻译弹窗提示离线
4. 存储容量到达上限给出提示

# UI基础规范
简洁轻量化，无多余动画；弹窗自适应深色/浅色网页；不强制遮挡页面内容
